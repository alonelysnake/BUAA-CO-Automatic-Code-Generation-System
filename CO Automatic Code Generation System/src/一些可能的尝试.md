对涉及分支指令的可能处理方式
--
1.所有指令跳转到最后生成label的区域，并以beq 0 0 end结束（jal用jr ra），每个label块内部运行不涉及跳转（目前逻辑）

缺点：未发生跳转的指令可以检测多个，发生的只能检测一个，只能检测正跳转

2.同1，label块内处理：label0作为负跳转时的标签，I类分支先正条再负跳（感觉可以尝试）

缺点：未发生跳转的指令可以检测多个，发生的只能检测一个
